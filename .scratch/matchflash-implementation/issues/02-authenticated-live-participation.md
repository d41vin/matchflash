# 02 — Authenticated live participation

**What to build:** Phantom/Convex identity mapping and a signed-in live reaction flow that records participation while keeping Match Room browsing anonymous.

**Blocked by:** 01 — Anonymous Match Room foundation.

**Status:** ready-for-agent

- [ ] Embedded and injected wallet sign-in resolve to one authenticated MatchFlash user without accepting a user identity from the client.
- [ ] A signed-in user can react during a live fixture, while an anonymous user receives a non-disruptive sign-in nudge.
- [ ] A live reaction records participation eligibility and a replay-only interaction cannot.
